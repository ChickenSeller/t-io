t-io1.7.0发布，3000行代码让天下没有难开发的即时通讯

这是一个历经磨难的版本，差点不能和大家在开源界见面了，不过值得庆幸的最终还是在开源界发布了。

t-io是基于jdk aio实现的易学易用、稳定耐操、性能强悍、内置功能丰富、代码只有3000多行的即时通讯框架。字母 t 取talent（天才）的首字母，也可以理解为"特快"，同时也是作者姓氏的首字母。一些传送门

  • 官 网
  • 开源中国收录地址：https://www.oschina.net/p/t-io
  • 码云地址：https://git.oschina.net/tywo45/t-io/
  • API（先看Aio.java的就好，然后就看本系列博客吧）
  • 资料及问题汇总

  1. 更新内容

    1. context的id改为uuid，为后面内置集群功能打下基础
    2. Aio.java增加n多便捷的方法，不一一列举，方法名即注释是t-io的一个基本原则，读过t-io的自然懂，
    3. 线程池部分代码精简，原来是引入了全部的talent系列线程池框架的代码，考虑到大家并不需要，所以精简到t-io必须的代码
    4. 本版本没有bug修复，但有几处日志去掉了，还是把打印权交给应用的开发人员

  2. 后继工作预告

    1. 规划企业版功能及服务，让企业用户可以放心使用，毕竟企业花了钱才用着更放心！
    2. 持续优化社区版（貌似没什么好优化的了，用户反映太稳定了，怎么操都不死，所以我不弄个企业版的出来，后面都没机会上开源中国的新闻了）
    3. 协助天蓬小猪完善t-io文档，不过用户们貌似并不需要文档，看完showcase就啥都会了（天蓬小猪是开源中国的官方人员，没想到t-io用3000多行代码就把开源中国的官方人员也成功吸粉了^_^）
    4. 规划t-io用户群体行为规范，在制度层面保障品行优良的用户获得更好的技术支持
